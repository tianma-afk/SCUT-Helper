# 商品模块接口文档

### 基础URL

```Plain
http://localhost:8000
```

## 概述

本文档详细描述了二手商品系统的商品模块 API 接口，覆盖「商品浏览（带搜索）」「商品详情」核心功能，为极简版本，仅满足基础展示需求。

## 获取商品分类列表

- **接口地址**: `GET /api/product/categories`

- **请求参数**:

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|       |         |     |                 |
| ----- | ------- | --- | --------------- |
| 参数名   | 类型      | 必填  | 说明              |
| skip  | integer | 否   | 跳过的记录数，默认为0     |
| limit | integer | 否   | 返回的记录数限制，默认为100 |

- **请求示例**:

```Plain
GET /api/product/categoriesGET /api/product/categories?skip=0&limit=10
```

- **响应示例**:

```JSON
{  "code": 200,  "message": "success",  "data":"商品分类列表"}
```

## 获取商品列表（搜索浏览界面）

- **接口地址**: `GET /api/product/list`

- **接口说明**: 分页获取上架商品列表，支持按商品名称模糊搜索，返回商品卡片所需的极简数据

- **请求参数**:

表格

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|          |         |     |                             |
| -------- | ------- | --- | --------------------------- |
| 参数名      | 类型      | 必填  | 说明                          |
| keyword  | string  | 是   | 搜索关键词（模糊匹配商品名称，为空时返回全部上架商品） |
| page     | integer | 否   | 页码，默认为 1                    |
| pageSize | integer | 否   | 每页显示的商品数量，最大值为 100，默认为 10   |

- **请求示例**:

plaintext

```Plain
GET /api/product/listGET /api/product/list?keyword=手机&page=2&pageSize=15
```

## 获取商品详情（详情界面）

- **接口地址**: `GET /api/product/detail`

- **接口说明**: 根据商品 ID 获取商品全部详细信息

- **请求参数**:

表格

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|     |         |     |       |
| --- | ------- | --- | ----- |
| 参数名 | 类型      | 必填  | 说明    |

## 提交商品（发布商品）

- 接口地址: `POST /api/product/add`

- 接口说明：发布新的二手商品，提交商品全部信息，成功后商品默认处于上架状态

- 请求参数: JSON 格式，POST 请求体传参

- 表格

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|                     |        |     |                   |
| ------------------- | ------ | --- | ----------------- |
| 参数名                 | 类型     | 必传  | 说明                |
| product_category_id | int    | 是   | 商品分类 ID           |
| product_name        | string | 是   | 商品名称              |
| product_price       | float  | 是   | 商品价格              |
| product_images      | string | 是   | 商品图片链接（单张 / 多张拼接） |
| product_content     | string | 是   | 商品描述（新旧、规格等）      |
| trade_desc          | string | 是   | 交易方式（自提 / 邮寄等）    |
| publisher_id        | int    | 是   | 发布者 ID            |

- 请求示例:

Plain Text

```Plain
 POST /api/product/add
```

```JSON
{    "product_category_id":1,      "product_name":"99 新平板",    "product_price":800,    "product_images":"https://example.com/pad.jpg",    "product_content":" 自用平板，无磕碰，功能正常 ",    "trade_desc":" 邮寄 + 自提 ",    "publisher_id":1} 
```

- 响应示例:

```JSON
{      "code": 200,     "message": "success",    "data": {            "product_id": 3              }} 
```

- 异常响应示例（参数缺失）:

```JSON
 {       "code": 400,      "message": "商品名称为必传参数",      "data": null } 
```

## 修改商品（编辑商品）

- 接口地址: `PUT /api/product/update`

- 接口说明：根据商品 ID 修改商品信息，支持单独修改部分字段，发布者仅可修改自身发布的商品

- 请求参数: JSON 格式，PUT 请求体传参

- 表格

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|                     |        |     |                     |
| ------------------- | ------ | --- | ------------------- |
| 参数名                 | 类型     | 必传  | 说明                  |
| product_id          | int    | 是   | 要修改的商品 ID           |
| product_category_id | int    | 否   | 商品分类 ID             |
| product_name        | string | 否   | 商品名称                |
| product_price       | float  | 否   | 商品价格                |
| product_images      | string | 否   | 商品图片链接（单张 / 多张拼接）   |
| product_content     | string | 否   | 商品描述（新旧、规格等）        |
| trade_desc          | string | 否   | 交易方式（自提 / 邮寄等）      |
| status              | int    | 否   | 商品状态（1 = 上架，0 = 下架） |

- 请求示例

plaintext

```Plain
 PUT /api/product/update
```

- 请求体：

```JSON
{    "product_id":3,    "product_price":750,    "trade_desc":"自提"} 
```

- 响应示例:

```JSON
{      "code": 200,     "message": "success",     "data": null}
```

- 异常响应示例:

```JSON
 {       "code": 404,     "message": "商品不存在",     "data": null}
```

```JSON
{      "code": 403,      "message": "无权限修改该商品",      "data": null} 
```

## 删除商品

- 接口地址: `DELETE /api/product/delete`

- 接口说明：根据商品 ID 删除商品，发布者仅可删除自身发布的商品，删除后商品数据不可恢复

- 请求参数: URL 传参

- 表格

<style> td {white-space:nowrap;border:0.5pt solid #dee0e3;font-size:10pt;font-style:normal;font-weight:normal;vertical-align:middle;word-break:normal;word-wrap:normal;}</style>

|              |     |     |                |
| ------------ | --- | --- | -------------- |
| 参数名          | 类型  | 必传  | 说明             |
| product_id   | int | 是   | 要删除的商品 ID      |
| publisher_id | int | 是   | 发布者 ID（接口鉴权使用） |

- 请求示例:

```Plain
DELETE /api/product/delete?product_id=3&publisher_id=1
```

- 响应示例:

```JSON
 {       "code": 200,       "message": "success",       "data": null } 
```

- 异常响应示例:

```JSON
 {       "code": 404,       "message": "商品不存在",       "data": null } {       "code": 403,       "message": "无权限删除该商品",       "data": null } 
```
